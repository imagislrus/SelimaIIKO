# iBOOKKEEPER

Система приёма и обработки товарных накладных с OCR-распознаванием.

**Продакшн:** https://ibookkeeper.org

## Стек

- **Backend:** Node.js, Express, MongoDB (Mongoose)
- **OCR:** Yandex Vision API (`/ocr/v1/recognizeText`)
- **AI-парсинг:** Claude Haiku (`claude-haiku-4-5-20251001`) — извлечение структурированных данных из OCR-текста
- **Процесс-менеджер:** PM2 (процесс `invoices`)
- **Веб-сервер:** Nginx + SSL (Let's Encrypt / Certbot)

## Структура проекта

```
server.js              — основной сервер (API, роуты, модели)
public/
  admin.html           — админ-панель (логин, накладные, контрагенты)
  upload.html          — страница загрузки для контрагентов (мобильная)
  uploads/             — загруженные фото накладных
.env                   — конфигурация (API-ключи, MongoDB URI, порт)
```

## Модели данных

### Supplier (Контрагент)
- `name`, `phone`, `token` (уникальная ссылка для загрузки)

### Invoice (Накладная)
- `supplier_id`, `supplier_name`, `received_at`
- `photos` — массив имён файлов
- `ocr_raw` — сырой текст OCR
- `items` — распознанные товары `[{ name, unit, qty, price, total }]`
- `total_sum`, `status` (`new` / `confirmed` / `rejected`)
- `payment_status` (`paid` / `not_paid` / `paid_cash` / `paid_delivery`)
- `payment_amount`
- `items_edited` — флаг ручного редактирования
- Сессионные поля: `session_id`, `session_index`, `session_total`, `session_confirmed`, `session_incomplete`, `session_received`

### Product (Товар)
- `name`, `unit`, `last_price`, `updated_at`
- Автоматически обновляется при загрузке накладных

## API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/upload?token=...` | Страница загрузки для контрагента |
| POST | `/api/upload?token=...` | Загрузка фото (multipart, OCR + AI-парсинг) |
| POST | `/api/upload/commit` | Подтверждение сессии загрузки |
| GET | `/api/invoices` | Список накладных (последние 100) |
| PATCH | `/api/invoices/:id` | Обновление накладной (статус, товары, сумма) |
| GET | `/api/invoices/export/csv` | Экспорт подтверждённых накладных в CSV |
| GET | `/api/suppliers` | Список контрагентов |
| POST | `/api/suppliers` | Создание контрагента (генерация токена) |
| POST | `/api/ocr` | Прямой OCR-запрос (base64 → текст + парсинг) |

## Функционал

### Страница загрузки (upload.html)
- Мобильный интерфейс для контрагентов
- Фото с камеры или из галереи
- **Сжатие фото** перед отправкой: макс. 1200px ширина, JPEG качество 0.6 (~100-300 КБ)
- Статус оплаты для каждого фото: Оплачено / Не оплачено / С кассы / С доставкой
- Поле суммы оплаты (необязательно)
- **Фоновая отправка** — фото начинает отправляться сразу после выбора статуса оплаты
- Retry при ошибке (3 попытки с интервалом 2 сек)
- Индикация на карточке: 🕐 Ожидает → ⏳ Отправляется → ✅ Отправлено / ❌ Ошибка
- PATCH-обновление суммы с debounce, если фото уже отправлено
- Сессионный контроль: `session_id` генерируется при первом выборе статуса
- Кнопка "Отправить" досылает ошибочные/ожидающие, обновляет `session_total`, делает commit

### Админ-панель (admin.html)
- **Логин:** защищён паролем
- **Две вкладки:** Накладные, Контрагенты

#### Вкладка "Накладные"
- Боковая панель с фильтрами: Новые / Подтверждённые / Отклонённые
- Фильтр по контрагенту (выпадающий список)
- Карточки накладных с бейджами статуса, оплаты, редактирования
- Бейдж неполной сессии: ⚠️ "X из Y"
- **Split-view:** фото слева (45%), данные справа
- **Зум фото:** скролл для масштаба (0.5x–5x), перетаскивание мышью, двойной клик для сброса
- **Поворот фото:** кнопки ↺ Влево / ↻ Вправо (сброс зума при повороте)
- Ограничение перетаскивания: фото не выходит за пределы контейнера
- **Таблица товаров:** редактируемая в режиме правки, кнопка "Добавить строку", авто-пересчёт
- **Действия:** Подтвердить / Исправить / Отклонить (расположены между таблицей и OCR-текстом)
- **Клавиатурные сочетания:** Enter — подтвердить, E/F2 — исправить, Delete/Backspace — отклонить
- Экспорт подтверждённых в CSV (кнопка в фильтре "Подтверждённые")
- Автообновление списка каждые 30 секунд

#### Вкладка "Контрагенты"
- Добавление контрагента (имя, телефон)
- Список с кнопками копирования ссылки для загрузки

### Сессионный контроль
- Каждая загрузка получает `session_id`
- После отправки всех фото — запрос `/api/upload/commit` сверяет количество
- Фоновая проверка каждую минуту: незакрытые сессии старше 15 минут помечаются как `session_incomplete`
- В админке: жёлтый бейдж на неполных сессиях

### OCR-пайплайн
1. Фото сжимается на клиенте (Canvas API)
2. Загружается на сервер (Multer)
3. Отправляется в Yandex Vision OCR (base64)
4. Распознанный текст парсится Claude Haiku в JSON
5. Результат сохраняется в MongoDB
6. Товары обновляются в коллекции Products

```bash
MONGODB_URI=mongodb://localhost:27017/invoices
PORT=3000
```

## Запуск

```bash
npm install
pm2 start server.js --name invoices
```
